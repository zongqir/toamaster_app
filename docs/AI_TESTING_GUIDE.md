# AI 解析功能测试指南

## 测试目的

验证默认文心AI和自定义AI配置是否正常工作。

## 测试环境

- 小程序或 H5 环境
- 网络连接正常

## 测试用例

### 测试 1：默认文心AI解析（无需配置）

**前置条件**：
- 未配置任何自定义AI
- 设置页面的"使用自定义 AI"开关为关闭状态

**测试步骤**：
1. 打开小程序，进入"导入会议流程"页面
2. 粘贴以下测试数据：
   ```
   19:00  开场致辞  2分钟  主持人
   19:02  即兴演讲  15分钟  即兴官
   19:17  备稿演讲  20分钟  备稿官
   19:37  总结  3分钟  主持人
   ```
3. 确保右上角显示"AI 解析"标签
4. 点击"智能解析"按钮
5. 等待解析完成

**预期结果**：
- 显示"AI 解析中..."加载提示
- 解析成功，跳转到时间轴预览页面
- 显示 4 个环节，时长和负责人正确解析
- 控制台日志显示"使用 AI 服务: 文心AI（默认）"

**失败处理**：
- 如果报错，检查控制台错误信息
- 确认 Edge Function 已正确部署
- 确认 INTEGRATIONS_API_KEY 已正确配置

---

### 测试 2：自定义AI解析（硅基流动）

**前置条件**：
- 拥有硅基流动账号和 API Key
- 账户有足够的额度

**测试步骤**：
1. 进入"设置"页面
2. 找到"AI 解析配置"区域
3. 开启"使用自定义 AI"开关
4. 填写配置：
   - API 地址：`https://api.siliconflow.cn/v1/chat/completions`
   - API Key：`sk-xxxxxxxxxxxxxxxx`（您的真实 API Key）
   - 模型名称：`Qwen/Qwen2.5-7B-Instruct`
5. 返回"导入会议流程"页面
6. 粘贴测试数据（同测试 1）
7. 点击"智能解析"按钮
8. 等待解析完成

**预期结果**：
- 解析成功，跳转到时间轴预览页面
- 控制台日志显示"使用 AI 服务: 自定义AI"
- 解析结果准确

**失败处理**：
- 检查 API Key 是否正确
- 检查账户额度是否充足
- 检查 API 地址是否正确
- 查看控制台错误信息

---

### 测试 3：切换AI服务

**测试步骤**：
1. 使用自定义AI解析一次（参考测试 2）
2. 进入"设置"页面
3. 关闭"使用自定义 AI"开关
4. 返回"导入会议流程"页面
5. 再次解析相同的数据
6. 验证使用默认文心AI

**预期结果**：
- 两次解析都成功
- 控制台日志显示不同的 AI 服务
- 解析结果一致

---

### 测试 4：本地解析（无需AI）

**测试步骤**：
1. 进入"导入会议流程"页面
2. 点击右上角切换到"本地解析"标签
3. 粘贴测试数据

- 快速解析完成（无网络请求）
- 跳转到时间轴预览页面
- 基本信息正确解析

## 常见错误及解决方案

### 错误 1：`Cannot read properties of undefined (reading 'model')`
**原因**：Edge Function 尝试访问不存在的 aiConfig 属性  
**解决**：已修复，重新部署 Edge Function

### 错误 2：`AI API 请求失败: 401`
**原因**：API Key 无效或过期  
**解决**：检查并更新 API Key

### 错误 3：`AI API 请求失败: 429`
**原因**：请求频率超限或额度不足  
**解决**：等待一段时间或充值账户

### 错误 4：`解析失败: 无法提取有效的 JSON`
**原因**：AI 返回的内容不是有效的 JSON  
**解决**：
- 检查表格格式是否正确
- 尝试使用本地解析
- 简化表格内容后重试
