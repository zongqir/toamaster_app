### AI 分组规则
AI 会根据会议流程自动识别并分组：
- 备稿演讲组（preparedSpeech）
- 备稿点评组（evaluation）
- 即兴演讲组（tableTopics）
- 会议官员组（时间官、语法官、哼哈官等）
- 其他角色组（SAA、主持人等）

### 投票规则（AI 自动计算）
- 每组候选人数 <= 3：maxSelections = 1
- 每组候选人数 4-6：maxSelections = 2
- 每组候选人数 >= 7：maxSelections = 3

### 防重复投票
- 使用设备指纹（浏览器/设备唯一标识）
- 同一设备只能投票一次

## 使用流程

### 1. 创建投票（时间官）
1. 在 Timeline 页面点击投票图标按钮
2. 自动跳转到投票编辑页面
3. **AI 自动分组**：系统调用文心一言 AI 分析会议流程并生成分组
4. **手动调整**：
   - 可以编辑分组名称
   - 可以调整最大可选数（1-3）
   - 可以新增/删除分组
   - 可以编辑候选人姓名和描述
   - 可以新增/删除候选人
   - 可以点击"重新 AI 分组"重新生成
5. 点击"保存并创建投票"
6. 显示投票 ID，有效期 24 小时

### 2. 参与投票（会议成员）
1. 打开小程序，点击底部"投票入口"标签
2. 自动加载当前活跃投票并进入
3. **首次投票**：
   - 为每个分组选择候选人
   - 提交投票（无需填写姓名）
4. **已投票用户**：
   - 自动显示之前的投票选择
   - 可以点击"修改投票"按钮修改选择
   - 可以点击"查看结果"查看投票结果
   - 修改后点击"保存修改"更新投票

### 3. 查看结果
- 投票后可以查看结果
- 时间官可在 Timeline 页面点击图表图标查看结果
- **会议复盘时**：在统计页面直接显示投票结果
  - 如果有投票，显示各分组的前三名和投票统计
  - 如果没有投票，显示"暂无投票"和"创建投票"按钮
- 支持导出结果文本

**解决方案**：
1. 创建 `CustomTabBar` 组件，模拟原生 tabBar 的外观和行为
2. 在投票页面和结果页面中引入该组件
3. 使用 `fixed bottom-0` 固定在页面底部
4. 页面容器添加 `pb-14` 为底部导航留出空间
5. ScrollView 底部添加额外占位空间
6. 点击 tab 使用 `Taro.switchTab` 跳转到 tabBar 页面

## 问题根源分析

### 投票结果为空的真正原因 ✅ 已彻底解决

**问题现象**：
- 用户已投票，但会议复盘显示"暂无投票"
- 投票结果显示 totalVoters: 0，所有分组 totalVotes: 0

**根本原因**：
1. 投票会话 ID 使用随机生成的 6 位字符串
2. 每次创建投票都生成新的会话 ID
3. 同一个会议创建了多个投票会话（9个）
4. 用户在旧的投票会话中投票，但系统查询时可能取到新的空会话
5. 导致数据混乱，无法正确关联投票记录

**完整解决方案**：
- ✅ **核心改动**：投票会话 ID = 会议 ID，分组 ID = `{会议ID}_group_{索引}`，候选人 ID = `{会议ID}_candidate_{分组索引}_{候选人索引}`
- ✅ 修改 vote-edit 页面，移除所有随机 ID 生成逻辑
- ✅ 使用 `currentSession.id` 作为投票会话 ID
- ✅ 分组和候选人 ID 使用会议 ID 作为前缀，加上索引号
- ✅ 修改 createVotingSession 方法，支持覆盖更新
- ✅ 简化 getVotingResultByMeetingId 方法，直接使用会议 ID 查询
- ✅ 删除数据库中所有旧的投票记录，使用新的 ID 规则

**ID 规则示例**（会议 ID = "abc123"）：
- 投票会话 ID: `abc123`
- 第一个分组 ID: `abc123_group_0`
- 第一个分组的第一个候选人 ID: `abc123_candidate_0_0`
- 第一个分组的第二个候选人 ID: `abc123_candidate_0_1`
- 第二个分组 ID: `abc123_group_1`
- 第二个分组的第一个候选人 ID: `abc123_candidate_1_0`
